**docker compose up**
Somit wird unser Webserver per Container gestartet.

**docker compose down**
Dies terminiert unseren Container Webserver wieder.

**docker image ls**
Hier kann man alle laufenden Docker Anwendungen sehen.

**Docker inspect <Tag/ID>**
Da kann man näher auf die laufenden Docker Anwendungen hinschauen und die Eigenschaften sehen.
